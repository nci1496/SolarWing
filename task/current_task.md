# 当前任务卡片

## 任务名称
Phase 4 首个子任务：三幅曲线绘图控件骨架

## 所属阶段
Phase 4：曲线可视化

## 当前任务背景
- Phase 1 已完成主窗口左右分栏骨架，左侧已有三块曲线区域占位。
- Phase 2 已完成基本设置与参数设置对话框骨架。
- Phase 3 已完成仿真核心、诊断日志与边界条件修复，仿真结果已经可以稳定输出时间、拉力、误差、目标位移、实际位移、速度等序列。
- 当前具备进入曲线可视化阶段的前提条件。

## 当前任务目标
- 将左侧三块占位区域升级为可绘制的曲线控件骨架。
- 为后续接入仿真数据与鼠标交互预留统一接口。
- 本轮优先完成“能显示坐标轴/标题/基础绘图区域”的结构，不要求一次完成所有交互细节。

## 本轮必须完成
1. 新建通用曲线绘图控件，例如：`widgets/plotwidget.*`。
2. 支持基础绘制能力：
   - 绘图区背景
   - 坐标轴
   - 标题
   - 简单折线序列绘制
3. 在主窗口左侧三块区域中接入三个绘图控件，分别对应：
   - 拉力曲线
   - 位移误差曲线
   - 位移曲线（实际/目标）
4. 为后续数据接入预留清晰接口，例如：
   - 设置单条曲线数据
   - 设置多条曲线数据
   - 清空曲线
5. 保持当前主窗口布局基本不变，不破坏已有菜单和对话框入口。

## 本轮明确不做
- 不要求完成鼠标十字光标。
- 不要求完成单击显示坐标点。
- 不要求完成双击清空坐标点。
- 不要求接入实时 `QTimer` 播放。
- 不要求完成右侧动画。
- 不要求完成参数保存读取。

## 建议涉及文件
- `widgets/plotwidget.h`
- `widgets/plotwidget.cpp`
- `mainwindow.h`
- `mainwindow.cpp`
- `mainwindow.ui`（如需替换占位控件）
- `SolarWing.pro`

## 推荐实现方式
- 使用自绘 `QWidget` 作为基础绘图控件，先完成可维护的最小骨架。
- 将坐标轴范围、标题、颜色、序列数据封装为成员变量或简单结构。
- 位移图建议从一开始就支持两条曲线，便于后续同时显示“实际位移/参考位移”。
- 若当前暂无真实仿真接线，可先保留空数据渲染逻辑，但控件应能正常显示边框、标题与坐标区域。

## 验收标准
- 左侧三块区域已不再只是静态占位，而是实际绘图控件。
- 至少能显示标题、坐标轴和基础绘图区域。
- 已具备后续接入仿真序列数据的接口。
- 不破坏现有主窗口布局与菜单功能。
