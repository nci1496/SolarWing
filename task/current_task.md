# 当前任务卡片

## 任务名称
二次目标异常修复任务（边界条件方向）

## 所属阶段
Phase 3：仿真层验证与诊断

## 上一轮诊断结论摘要
- 已为二次目标异常场景建立专用诊断入口：`simulation/test_simulation_diagnostics.*`。
- 已输出关键时刻日志：`t=0.0`、`0.1`、`0.5`、`1.0`、`2.0`、`4.0`。
- 每个快照均包含：`target`、`actual`、`velocity`、`acceleration`、`tension`、`error`。
- 诊断结论当前归类为 **C. 边界条件**。
- 判断依据：二次目标值随时间正常增长，PID 张力保持非负输出，但实际位移被钳制在 `0` 附近时，速度仍持续为负，说明异常更接近“位移边界处理与状态更新不一致”而不是目标生成本身。

## 当前任务背景
- 当前仿真链路与常值目标测试基本正常。
- 二次目标异常已经通过日志手段定位到边界条件方向。
- `SolarWingSimulator::step()` 当前会在积分后执行 `m_state.displacement = qMax(0.0, m_state.displacement);`，但未同步处理负向速度状态。
- 这会导致位移被强制截断后，速度仍保留负值，形成持续向下的动力学趋势，并在二次目标场景中放大问题。

## 当前任务目标
- 在不改变目标生成与 PID 参数的前提下，修复“位移触底后速度状态未同步”的仿真边界条件问题。
- 保持常值目标场景不被破坏。
- 修复后重新验证二次目标场景，确认不再出现“位移为 0 且速度持续大负值”的异常。

## 本轮必须完成
1. 检查并修复 `SolarWingSimulator` 中与下边界相关的状态处理。
2. 修复策略必须聚焦仿真边界一致性，不修改 PID 参数与动力学公式主体。
3. 保留现有诊断入口，便于修复后复测。
4. 至少复测：
   - `test_simulation`
   - `test_pid_tuning`

   - 目标是否继续正常增长；
   - 位移是否仍长期锁死在 0；
   - 速度是否仍持续为明显负值。
