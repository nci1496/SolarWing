# 当前任务卡片

## 任务名称
右侧动画区信息增强与比例映射修正

## 所属阶段
Phase 7：参数保存读取与验收（收尾增强）

## 当前背景
- `StepTest.ini` 与 `SinTest.ini` 已可打开并复现实验场景
- `StepTest` 当前效果基本满意
- `SinTest` 目前仍基于“二次目标”能力组织，暂不强行扩展为严格正弦目标
- 当前最明显的可用性短板集中在右侧动画区的信息表达与比例映射

## 当前任务目标
- 提升右侧动画区的可读性与演示体验
- 让动画显示比例与当前 `Limit` 真正一致
- 为最终录屏和课程验收提供更直观的数值反馈

## 本轮优先要做
1. 在右侧动画区显式显示 `Target=xx.xx`
2. 在右侧动画区显式显示 `Actual=xx.xx`
3. 在右侧动画区显式显示 `Limit=xx.xx`
4. 在右侧动画区显式显示 `Error=xx.xx`
5. 在动画区域下方显示 `t=xx.xx / xx.xx`
6. 增加可拖拽的播放进度条
7. 修正位移动画按固定像素映射的问题，改为基于当前 `Limit` 自适应缩放
8. 在右侧增加 `0 ~ Limit` 刻度尺

## 当前明确不做
- 暂不新增真正的 `Sinusoidal` / 正弦目标类型
- 暂不重做整体界面风格
- 暂不扩展与本轮无关的仿真模型能力

## 当前问题定义
- 目前 `AnimationWidget` 的位移到像素映射近似固定，因此当 `Limit` 从较小值继续增大时，动画中的 `Limit` 参考线和太阳翼可运动空间会过早“顶满”
- 这会导致 `3.8 ~ 10.0` 一段内，虽然参数数值仍在变大，但动画视觉位置几乎不再变化，影响理解和演示
- 正确做法应是：右侧动画区将 `Limit` 作为当前量程上限，按“可绘制高度 / Limit”动态计算位移映射比例

## 建议实现方向
1. 由主窗口在播放过程中把当前 `target / actual / error / time / totalTime / limit` 传给动画区
2. `AnimationWidget` 负责绘制实时数值文本、刻度尺和自适应比例动画
3. 进度条支持显示当前播放进度，并允许拖拽跳转到指定样本
4. 若时间允许，再补状态栏联动显示当前播放时间与误差

## 验收标准
- 右侧动画区可实时看到 `Target`、`Actual`、`Limit`、`Error` 数值
- 动画区底部可实时看到 `t=当前时间 / 总时长`
- 进度条可随播放推进，并支持手动拖拽跳转
- 当 `Limit` 取 `0.1 ~ 10.0` 不同值时，动画显示比例都能合理自适应
- 右侧刻度尺与 `Limit` 上限一致，不再出现高区间视觉不变化的问题
