# SolarWing 课程验收流程

> 配套课程要求：**实现太阳翼展开时变重力卸载控制的可视化编程与仿真**。本流程按"先核心功能、再两个场景"组织，所有截图供 `report/main.tex` 与现场汇报引用。

---

## 0. 全局前提

- 在 Qt Creator 中打开 `SolarWing.pro`，构建（默认 Desktop Qt 套件）并运行 SolarWing
- 程序启动后默认尺寸约 1200×750；标题栏显示 "SolarWing"
- 截图工具：Windows 自带截图工具（`Win+Shift+S` 选区域）或任意截图工具
- 截图保存路径：本仓库 `report/image/`
- 截图命名约定：`step<两位编号>_<主题>.png`，例如 `step01_main_window.png`
- 截图比例：尽量与窗口实际尺寸一致，避免引入黑边
- 本流程不修改任何源码 / `.pro` / `.ini`；如发现非预期行为，回到对应模块迭代

---

## 1. 关键参考

- 仓库根：`StepTest.ini` / `SinTest.ini`（验收场景源，不要改动）
- 菜单项（位于「设置」「控制」两个菜单）：
  - 设置：基本设置 / 参数设置 / 保存配置 / 打开配置
  - 控制：开始 / 暂停 / 停止
- 快捷键：
  - `Ctrl+Alt+F` 基本设置
  - `Ctrl+Alt+P` 参数设置
  - `Alt+S` 开始
  - `Alt+P` 暂停
  - `Alt+T` 停止
- 上限报警标题：`**上限报警**`；文本：「太阳翼位置已达到设置的最高值，仿真已停止。」

## 3. 详细步骤

> 每一步包含「操作 → 预期结果 → 截图文件名 → 备注」四个字段。

### A. 启动与主界面

#### step01：主窗口初始布局
- **操作**：运行 SolarWing，进入默认主界面
- **预期结果**：主窗口 1200×750；左侧 2/3 三幅曲线（拉力曲线、位移误差曲线、位移曲线），右侧 1/3 动画区（卷扬机 / 钢丝绳 / 太阳翼静态示意）；菜单栏含「设置」「控制」；状态栏显示默认提示
- **截图**：`report/image/step01_main_window.png`
- **备注**：默认参数（PID：Kp=100, Ki=25, Kd=45；目标类型=二次曲线；maxDisplacement=1.5 m），首次启动会执行一次仿真并刷新曲线

### B. 基本设置

#### step02：基本设置对话框（含显示样式与积分算法）
- **操作**：菜单「设置」→「基本设置」（或 `Ctrl+Alt+F`）
- **预期结果**：弹出"基本设置"对话框；可见曲线绘图区背景色、文字色、字体、字号控件，以及"积分算法"单选（欧拉法 / 二阶龙格-库塔法 / 四阶龙格-库塔法）
- **截图**：`report/image/step02_basic_settings.png`
- **备注**：建议把积分算法切到「四阶龙格-库塔法」后点确认；如要看背景色与字体的实际生效，对比 step01 与 step04

### C. 参数设置

#### step03：参数设置对话框（默认二次曲线）
- **操作**：菜单「设置」→「参数设置」（或 `Ctrl+Alt+P`）
- **预期结果**：弹出"参数设置"对话框；分五个区：系统动力学参数（k/g/m1/θ）、控制器参数（Kp/Ki/Kd 滑块 + 输入框）、目标设置（定值 / 时变二次曲线，A、B）、仿真参数（仿真时间、步长）、调试操作区（开始/暂停/停止 按钮 + 指示灯）；底部"最高位移"输入框
- **截图**：`report/image/step03_param_settings.png`
- **备注**：此处保留默认参数即可，无需改动；记录下当前最高位移（如 1.5 m）以便在 StepTest/SinTest 中对照

#### step04：参数设置对话框（切换定值目标 + 调高 PID）
- **操作**：在"参数设置"对话框里，目标设置区选择"定值 l_d"；最高位移设为 "2.5"；点击"确认"
- **预期结果**：曲线三幅图与右侧动画按新参数刷新；状态栏不报错
- **截图**：`report/image/step04_param_constant_target.png`
- **备注**：step04 与 step03 一并展示"参数可设置 + 设置后自动刷新"这一评分点；如发现曲线与动画未变，重新点一次确认

### D. 自动刷新

> step04 已覆盖曲线与动画区在参数改动后自动刷新；如希望单独保留一张"确认前 → 确认后"对照，复制 step04 命名为 `step04b_param_after_refresh.png` 即可。

### E. 快捷键

- **操作**：`Ctrl+Alt+F` / `Ctrl+Alt+P` 各自弹窗已隐含在 step02、step03 中
- **预期结果**：不新增截图
- **备注**：若需独立证据，可在主窗口聚焦时按下 `Alt+P`（暂停，当前无播放会显示状态栏提示）并截图 `step05_shortcut_alt_p.png`

### F. 播放控制（菜单 + 快捷键）

#### step05：开始 → 暂停 → 继续 → 停止
- **操作**：依次菜单「控制」→「开始」（或 `Alt+S`），等待 1~2 秒后「控制」→「暂停」（或 `Alt+P`），再「开始」（或 `Alt+S`），最后「停止」（或 `Alt+T`）
- **预期结果**：状态栏依次出现"开始/暂停/停止"提示；曲线与动画区随样本推进；停止后样本回到 0
- **截图**：`report/image/step05_playback_control.png`
- **备注**：建议在 step04 的"定值目标"参数下播放 2~3 秒后暂停再截，能同时显示曲线已开始抬升与动画区太阳翼移动

### G. 曲线交互

#### step06：十字光标 + 单击标点
- **操作**：把鼠标移到任一曲线图（建议位移曲线）上 → 出现十字光标 → 在曲线上某点**左键单击**
- **预期结果**：光标变十字；该位置出现标记点与坐标文本
- **截图**：`report/image/step06_crosshair_and_marker.png`
- **备注**：单击时鼠标不要拖动，否则只触发平移

#### step07：双击清空 + 滚轮缩放 / 拖拽平移
- **操作**：在 step06 同一图上**左键双击**（清除所有标记）→ 滚轮向上滚动 1~2 格（纵向放大）→ 鼠标按下并向右拖动 50 像素（横向平移）
- **预期结果**：标记点消失；坐标轴刻度等比变化（缩放）；曲线整体被平移
- **截图**：`report/image/step07_doubleclick_and_zoom.png`
- **备注**：若想同时体现缩放与平移，截图需在"已完成拖拽、松开鼠标后"再截

### H. 加载 StepTest

#### step08：打开配置 → StepTest.ini
- **操作**：菜单「设置」→「打开配置」→ 在文件对话框中选择仓库根目录的 `StepTest.ini` → 打开
- **预期结果**：曲线三幅图与右侧动画立即刷新为定值目标下的阶跃响应；动画区出现"目标位置"红色虚线参考
- **截图**：`report/image/step08_load_steptest.png`
- **备注**：若曲线变化不明显，是因为 StepTest 与默认值接近；可与 step04 状态对比

### I. StepTest 播放中段

#### step09：StepTest 播放中段
- **操作**：`Alt+S` 开始播放 → 等待 1.5~2.5 秒（播放会自动跑，可看曲线逐渐爬升）→ 截图
- **预期结果**：三幅曲线均已绘出有效轨迹；动画区太阳翼抬升到约最高位移的一半；右侧信息条 `Actual / Target / Error / Limit` 在变
- **截图**：`report/image/step09_steptest_playing.png`
- **备注**：建议在曲线已形成明显形状但**尚未触达上限**时截；可用 Alt+P 暂停再截更稳

### J. StepTest 上限报警

#### step10：上限报警对话框
- **操作**：继续保持播放或恢复播放；等到动画区太阳翼（实际位移）触达或越过红色"上限位置"参考线 → 弹出"上限报警"对话框
- **预期结果**：弹出标题"上限报警"、内容"太阳翼位置已达到设置的最高值，仿真已停止。"的 `QMessageBox`；点"确定"后播放自动停止
- **截图**：`report/image/step10_limit_alert.png`
- **备注**：若等待较久，可临时把 StepTest.ini 的 `maxDisplacement` 调小到 0.5~0.8 后再加载（注意：仅观察用，**不要保存覆盖**仓库根的 StepTest.ini）；也可以用 Alt+S → Alt+P 反复试探

### K. 加载 SinTest

#### step11：打开配置 → SinTest.ini
- **操作**：菜单「设置」→「打开配置」→ 选择仓库根目录的 `SinTest.ini` → 打开
- **预期结果**：曲线三幅图与右侧动画立即刷新为"时变二次曲线"目标的演示场景；动画区右侧信息条 `Target` 数值随时间递增
- **截图**：`report/image/step11_load_sintest.png`
- **备注**：SinTest 当前实现仍基于"二次目标"能力组织（见 `task/task_history.md` 2026-06-10 记录），不是严格正弦；课程允许二次目标则可继续，否则可口头说明

### L. SinTest 播放中段

#### step12：SinTest 播放中段
- **操作**：`Alt+S` 开始播放 → 等待 1.5~2.5 秒（建议在目标已抬升但未触达 Limit 时暂停）→ 截图
- **预期结果**：位移图上"参考位移（目标）"与"实际位移"两条曲线呈发散-收敛-跟随形态；拉力图在 0~5000 N 区间波动；误差图整体趋于 0
- **截图**：`report/image/step12_sintest_playing.png`
- **备注**：暂停（Alt+P）状态下截图最稳

### M. SinTest 动画区细节

#### step13：动画区四项数值 + 时间 + 进度条 + 刻度尺
- **操作**：保持 step12 的暂停状态；右侧动画区已显示 `Target / Actual / Limit / Error`、下方 `t=xx.xx / xx.xx`、底部进度条、右侧 `0~Limit` 刻度尺
- **预期结果**：4 项数值实时反映当前样本；时间与总时长显示一致；进度条位置与播放进度一致；刻度尺上限 = 当前 Limit
- **截图**：`report/image/step13_animation_panel.png`
- **备注**：可同时把窗口右侧调到最大，方便整张截图

### N. 保存 / 重新打开配置

#### step14：保存配置为 StepTest 副本
- **操作**：菜单「设置」→「保存配置」→ 文件名改为 `MyStepTest.ini`（避免覆盖仓库根的 StepTest.ini）→ 保存
- **预期结果**：弹出文件保存对话框，文件实际写入；无错误提示
- **截图**：`report/image/step14_save_config.png`

#### step15：重新打开配置验证复现
- **操作**：菜单「设置」→「打开配置」→ 选择刚保存的 `MyStepTest.ini` → 打开
- **预期结果**：曲线与动画区与 step11/step12 状态一致（或与保存时一致），证明保存/打开配置闭环生效
- **截图**：`report/image/step15_reload_config.png`
- **备注**：可与 step11 / step12 做"复现性"对照

### O. 退出

- **操作**：菜单「文件」或主窗口关闭按钮
- **预期结果**：不新增截图
- **备注**：截图步骤已完成
